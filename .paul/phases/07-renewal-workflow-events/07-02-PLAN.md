<objective>
## Goal
Add event timeline UI and renewal workflow UI to the license detail page, consuming the backend API endpoints built in Plan 07-01.

## Purpose
Transforms license events from API-only data into visible history and actionable renewal workflow — the user-facing half of Phase 7. Without this, events and renewal are CLI/API-only with no web visibility.

## Output
- Event timeline section on license detail page (chronological event history with fee display)
- "Renew License" button + modal for editors (sets new expiration date, optional fee)
- "Add Event" button + modal for editors (manual event recording)
- Route handler updated to pass events to template
- Tests for new UI-serving behavior
</objective>

## AC-2: Renewal Modal Works
```gherkin
Given I am an editor viewing a license detail page
When I click the "Renew" button
Then a modal opens with fields: new expiration date (required), fee amount (optional), fee type (optional), notes (optional)

```

## AC-3: Add Event Modal Works
```gherkin
Given I am an editor viewing a license detail page

```

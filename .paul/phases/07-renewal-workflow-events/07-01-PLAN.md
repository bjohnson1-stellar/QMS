<objective>
## Goal
Add the `license_events` table, event CRUD functions, fee tracking, auto-expire CLI job, and renewal API endpoint — the backend foundation for renewal workflow and event history.

## Purpose
Transforms static expiration dates into an active event-driven system. Every significant license action (issued, renewed, suspended, etc.) is recorded with optional fee tracking. The auto-expire job ensures licenses past their expiration date are marked expired automatically. This is the data layer that Phase 7 Plan 02 (UI) and Phase 8 (notifications) build on.

## Output
- `license_events` table with 7 event types and fee tracking
- DB functions: create_event, get_license_events, auto_expire_licenses
- API endpoints: GET/POST events, POST renew license
- CLI command: `qms licenses auto-expire`
- Migration with backfill of 'issued' events for existing licenses
- Tests covering event CRUD, auto-expire logic, renewal flow, validation
</objective>

    **Schema (licenses/schema.sql):**
    Add license_events table definition after the existing tables:
    ```sql
    CREATE TABLE IF NOT EXISTS license_events (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        license_id INTEGER NOT NULL REFERENCES state_licenses(id) ON DELETE CASCADE,
        event_type TEXT NOT NULL CHECK(event_type IN ('issued','renewed','amended','suspended','revoked','expired','reinstated')),
        event_date TEXT NOT NULL,
        notes TEXT,
        fee_amount REAL,
        fee_type TEXT CHECK(fee_type IS NULL OR fee_type IN ('application','renewal','amendment','late_fee','other')),
        created_by TEXT DEFAULT 'system',
        created_at TEXT DEFAULT (datetime('now'))
    );
    CREATE INDEX IF NOT EXISTS idx_license_events_license ON license_events(license_id);
    CREATE INDEX IF NOT EXISTS idx_license_events_type ON license_events(event_type);
    ```

    **Migration (licenses/migrations.py):**
    Add a new migration function (Phase 7 block) that:
    1. Creates license_events table (idempotent with IF NOT EXISTS)
    2. Creates indexes
    3. Backfills 'issued' events for existing licenses that have an issued_date (INSERT INTO license_events SELECT ... FROM state_licenses WHERE issued_date IS NOT NULL AND NOT EXISTS subquery to avoid duplicates on re-run)
    Wire into the migrate() function following existing pattern.

    **DB functions (licenses/db.py):**
    Add these functions following existing patterns (get_db(), _audit(), pagination dict):

    1. `create_event(license_id, event_type, event_date, notes=None, fee_amount=None, fee_type=None, created_by='system')`:
       - Validate license_id exists (SELECT 1 FROM state_licenses)
       - INSERT into license_events
       - Call _audit(conn, 'license_event', new_id, 'created', new_values={...})
       - Return the new event dict

    2. `get_license_events(license_id, limit=50)`:
       - SELECT * FROM license_events WHERE license_id=? ORDER BY event_date DESC, created_at DESC LIMIT ?
       - Return list of event dicts

    3. `auto_expire_licenses(dry_run=False)`:
       - SELECT id, holder_name, license_number, state_code, expiration_date FROM state_licenses WHERE status='active' AND expiration_date IS NOT NULL AND expiration_date < date('now')
       - If dry_run: return list without changes
       - For each: UPDATE status='expired', create_event(type='expired', date=today), _audit the status change
       - Return dict: {expired_count: N, licenses: [{id, holder_name, license_number, state_code, expiration_date}]}

    4. `renew_license(license_id, new_expiration_date, fee_amount=None, fee_type=None, notes=None, created_by='system')`:
       - Get current license (error if not found)
       - If current status is 'expired': update status to 'active', create 'reinstated' event first
       - Update expiration_date to new_expiration_date
       - Create 'renewed' event with fee info
       - _audit both the license update and event creation
       - Return updated license dict

    Avoid:
    - Do NOT modify existing functions (list_licenses, get_license, etc.)
    - Do NOT change the state_licenses status CHECK constraint
    - Do NOT add event creation inside existing update_license() — events are explicit actions

    import typer
    app = typer.Typer(help="License compliance management")

    @app.command()
    def auto_expire(dry_run: bool = typer.Option(False, "--dry-run", help="Show what would expire without changing")):
        from qms.licenses.db import auto_expire_licenses
        result = auto_expire_licenses(dry_run=dry_run)
        if dry_run:
            typer.echo(f"Would expire {len(result['licenses'])} licenses:")
            for lic in result['licenses']:
                typer.echo(f"  - {lic['holder_name']} ({lic['state_code']}) #{lic['license_number']} — expired {lic['expiration_date']}")
        else:
            typer.echo(f"Expired {result['expired_count']} licenses.")

                typer.echo(f"  - {lic['holder_name']} ({lic['state_code']}) #{lic['license_number']}")

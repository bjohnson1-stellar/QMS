## Goal
Add regulatory intelligence UI: compliance score on license detail pages, state requirements CRUD on state detail pages, and a compliance overview dashboard on the main licenses page.

## Purpose
Makes the 11-01 backend data visible and actionable — users can see compliance scores, manage state requirements, and identify gaps across all licenses from the UI.

## Output
- Compliance score card on license detail page (score gauge, met/unmet breakdown)
- State requirements table + CRUD modal on state detail page
- Compliance overview tab/section on main licenses page (gap analysis summary by state)
</objective>

<acceptance_criteria>

## AC-1: Compliance Score on License Detail
```gherkin
Given a license detail page for a license with state requirements defined
When the page loads
Then a "Compliance" card displays the compliance score (0-100) as a circular gauge
And lists met requirements with green checkmarks
And lists unmet requirements with red X marks and the reason
And if no requirements exist for this state/type, shows "No requirements defined"
```

## AC-2: State Requirements CRUD on State Detail
```gherkin
Given a state detail page (e.g., /licenses/state/OH)
When the page loads
Then a "State Requirements" section shows all requirements for that state in a table
And editors can add/edit/delete requirements via modal
And the table shows: license_type, requirement_type, description, fee, frequency
And changes persist via the existing /api/requirements endpoints
```

## AC-3: Compliance Overview on Licenses Page
```gherkin
Given the main licenses list page
When the page loads
Then a "Compliance" section shows per-state compliance summary
And each state row shows: state name, total licenses, avg score, fully compliant count
And states with gaps are highlighted in warning color
And clicking a state navigates to the state detail page
```

<objective>
## Goal
Create the regulatory intelligence database backend — schema, DB functions, API endpoints, and seed data for state licensing requirements, fee schedules, and compliance scoring.

## Purpose
This is the "Harbor Compliance Core" equivalent — a structured database of what each state requires for each license type (fees, renewal frequency, CE hours, bonds, insurance, exams). Enables compliance gap analysis and scoring per license.

## Output
- `state_requirements` table with full CRUD
- DB functions for requirements CRUD, compliance scoring, gap analysis
- API endpoints for requirements management and compliance reports
- CLI command to seed initial requirements data
</objective>

## AC-1: State Requirements Schema
```gherkin
Given the database is initialized
When I query the state_requirements table
Then it exists with columns: id, state_code, license_type, requirement_type, description, fee_amount, fee_frequency, renewal_period_months, authority_name, authority_url, effective_date, notes, created_at, updated_at, created_by
And requirement_type is constrained to: initial_application, renewal, ce_requirement, bond, insurance, exam, background_check, fingerprinting
And there is a UNIQUE constraint on (state_code, license_type, requirement_type)
```

## AC-2: Requirements CRUD API
```gherkin
Given a user with license module access
When they POST/GET/PUT/DELETE to /licenses/api/requirements endpoints
Then requirements are created, listed, retrieved, updated, and deleted
And all mutations are audit-logged
And list supports filtering by state_code and license_type
```

## AC-3: Compliance Score Calculation
```gherkin
Given a license with state_code='OH' and license_type='Contractor'
And state_requirements exist for OH/Contractor (renewal, bond, insurance)
When I request the compliance score for that license
Then the API returns { score: N, total_requirements: M, met: [...], unmet: [...] }
Where score = (met / total) * 100
And each requirement is marked met/unmet based on: active status, valid documents, CE credits
```

## AC-4: Gap Analysis
```gherkin
Given multiple licenses across states
When I request the gap analysis endpoint
Then it returns per-license breakdown of requirements vs what's on file
And groups results by state for overview
And identifies missing requirements (no matching document, expired bond, etc.)
```

## AC-5: Seed Data CLI
```gherkin
Given the state_requirements table is empty
When I run `qms licenses seed-requirements`
Then it populates requirements for states where we have active licenses
And logs the count of seeded rows
And is idempotent (running twice doesn't duplicate)
```

<task type="auto">
  <name>Task 2: DB functions for requirements CRUD, compliance scoring, and gap analysis</name>
  <files>licenses/db.py</files>
  <action>
    Add functions to licenses/db.py following existing patterns (_audit, generate_uuid, pagination dict):

    **Requirements CRUD:**
    - `list_requirements(conn, *, state_code=None, license_type=None, requirement_type=None, page=0, per_page=0)` → paginated dict
    - `get_requirement(conn, requirement_id)` → dict or None
    - `create_requirement(conn, data, created_by='system')` → dict (with _audit)
    - `update_requirement(conn, requirement_id, data, changed_by='system')` → dict (with _audit)
    - `delete_requirement(conn, requirement_id, deleted_by='system')` → bool (with _audit)
    - `get_requirements_for_license(conn, state_code, license_type)` → list of dicts

    **Compliance Scoring:**
    - `calculate_compliance_score(conn, license_id)` → dict with:
      - score (0-100), total_requirements, met_count
      - met: list of {requirement_id, requirement_type, description, evidence}
      - unmet: list of {requirement_id, requirement_type, description, reason}
      - Logic:
        - 'renewal': met if license status is 'active' and not expired
        - 'ce_requirement': met if CE credits >= required hours (use existing ce_requirements/ce_credits)
        - 'bond': met if license_documents has active doc_type='bond'
        - 'insurance': met if license_documents has active doc_type='insurance'
        - 'exam', 'background_check', 'fingerprinting': met if a note or event references it (simple keyword match), or a document exists
        - 'initial_application': always met for existing licenses (they were issued)

    **Gap Analysis:**
    - `get_compliance_gap_analysis(conn)` → list of dicts, one per license:
      - license_id, state_code, license_type, holder_name, score, total, met, unmet_list
      - Calls calculate_compliance_score for each active license
    - `get_compliance_summary_by_state(conn)` → dict keyed by state_code:
      - {state_code: {total_licenses, avg_score, fully_compliant, has_gaps}}

    **Seed data helper:**
    - `seed_state_requirements(conn, requirements_data, created_by='system')` → int (count seeded)
      - Uses INSERT OR IGNORE for idempotency (UNIQUE constraint handles dedup)
  </action>
  <verify>Write quick smoke test: create a requirement, retrieve it, calculate compliance for a license. Check _audit was called.</verify>
  <done>AC-3 and AC-4 satisfied: compliance scoring and gap analysis functions work correctly</done>

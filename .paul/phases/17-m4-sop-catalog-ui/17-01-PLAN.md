<objective>
## Goal
Build the M4 SOP Catalog UI tab — category-based SOP browser with 15 categories, search/filter within M4, scope tag display, and empty-state handling for unpopulated categories.

## Purpose
Completes the browsable quality manual interface (M1–M4). Users can navigate 15 SOP categories, view SOPs within each, and drill into individual SOP details — mirroring the M3 Programs pattern but adapted for document catalog browsing.

## Output
- M4 tab shows a grid of 15 category cards with SOP counts
- Clicking a category shows the SOP list within it (with status badges, scope tags)
- Clicking an SOP shows full detail (metadata, summary, scope tags, linked programs, history)
- Search bar on M4 tab searches across all SOPs
- Empty states for categories with no SOPs
- All navigation is in-page (no page reloads), with back buttons
</objective>

<acceptance_criteria>

## AC-1: Category Cards Grid
```gherkin
Given the user clicks the M4 tab
When the tab activates
Then a grid of 15 category cards renders, each showing:
  - Category code (e.g. "4.01") as a badge
  - Category name
  - Description (truncated if long)
  - SOP count (from API)
  - Parent program name (if linked)
And cards with 0 SOPs show a muted "No SOPs yet" indicator
```

## AC-2: Category Detail / SOP List
```gherkin
Given the user clicks a category card
When the category detail loads
Then the grid hides and a detail view appears showing:
  - Back button to return to category grid
  - Category header (code + name + description)
  - List of SOPs in that category (from /api/sops?category_id=N)
  - Each SOP row shows: document_id, title, status badge, version
  - Empty state if category has no SOPs ("No procedures in this category yet")
And clicking an SOP row navigates to SOP detail
```

## AC-3: SOP Detail View
```gherkin
Given the user clicks an SOP in the list
When the SOP detail loads
Then a detail view appears showing:
  - Back button to return to SOP list
  - Document ID badge + title
  - Status badge (draft/approved/published with color coding)
  - Version + revision
  - Scope tags as pills (if any)
  - Summary text (if available)
  - Linked programs list (if any)
  - Effective date, review cycle, approval info
  - Empty states for missing optional fields
```

## AC-4: M4 Search Integration
```gherkin
Given the user is on the M4 tab
When they type in the search bar and press Enter
Then the search calls /api/sops/search?q=<query>
And results replace the category grid showing matching SOPs
And each result shows document_id, title, category name, status
And a "Back to categories" link returns to the grid
And empty search results show "No SOPs matching '<query>'"
```

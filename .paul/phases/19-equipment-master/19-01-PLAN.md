<task type="auto">
  <name>Task 2: Build reconciliation engine and equipment CRUD</name>
  <files>pipeline/reconciler.py, pipeline/equipment.py</files>
  <action>
    Create pipeline/equipment.py with CRUD functions:
    - get_equipment_instance(project_id, tag) → dict or None
    - list_equipment(project_id, filters={}) → list of dicts
    - create_equipment_instance(project_id, tag, type_id, attrs) → id
    - update_equipment_instance(instance_id, attrs) → bool
    - get_equipment_appearances(instance_id) → list of dicts
    - get_equipment_relationships(project_id, tag) → list of dicts
    - create_system(project_id, system_tag, system_name, category, discipline) → id
    - assign_to_system(instance_id, system_id) → bool
    - advance_lifecycle(instance_id, new_stage, changed_by, evidence_doc_id, notes) → bool

    Create pipeline/reconciler.py with the reconciliation engine:

    reconcile_project(project_id, dry_run=False) → ReconcileResult:
      1. Query all unique equipment tags from pipeline extraction tables:
         - equipment table (tag column)
         - electrical_panels, electrical_transformers, electrical_switchgear,
           electrical_motors, electrical_disconnects (tag columns)
         - mechanical_equipment, utility_equipment, refrigeration_equipment,
           fire_protection_equipment (tag columns)
         - plumbing_fixtures (type_id as tag)
         Group by tag, collect all disciplines and sheets where each appears.

      2. For each unique tag:
         a. Check if equipment_instances record exists (upsert pattern)
         b. Determine equipment_type from the extraction data (infer from description/type fields)
         c. Create/update equipment_instances with best available attributes
            (prefer schedule sheets over plan sheets for attribute values)
         d. Create equipment_appearances for each discipline/sheet combination
         e. Extract attributes_on_sheet JSON from discipline-specific extraction data

      3. Build equipment_systems:
         - Group RCU-N + associated RAHU-N units into Refrigeration Systems
         - Group panels by electrical service (1xxx = Svc 1, 2xxx = Svc 2, 3xxx = Svc 3)
         - Group AHU + associated VAV/FPB into HVAC systems
         - Use tag prefix patterns and relationship data from panel schedules

      4. Build equipment_relationships:
         - Electrical: Panel feeds Equipment (from panel schedule circuit data)
         - Refrigeration: RCU serves RAHU (from P&ID piping connections)
         - Mechanical: AHU serves VAV/FPB (from VAV schedule associated_unit field)
         - Plumbing: Pump serves system (from piping connections)

      5. Create/find equipment_types:
         - Group instances by (manufacturer, model_base) or by equipment_type string
         - Set expected_disciplines based on equipment category:
           Condensing Unit → [Refrigeration, Electrical, Structural]
           AHU → [Mechanical, Electrical, Structural, Controls]
           Panel → [Electrical]
           Pump → [Mechanical/Plumbing, Electrical]
           Boiler → [Mechanical, Electrical, Plumbing, Structural]
           Transformer → [Electrical]
           Exhaust Fan → [Mechanical, Electrical]

      6. Return ReconcileResult with counts: instances, appearances, systems,
         relationships, types, duration_ms

    Handle idempotency: Use UPSERT (INSERT OR REPLACE) keyed on
    (project_id, tag) for instances, (instance_id, discipline, sheet_id) for appearances.

    Avoid: Modifying any existing extraction tables. The reconciler is read-only
    on extraction data, write-only to equipment registry tables.
  </action>

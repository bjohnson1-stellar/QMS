<objective>
## Goal
Add primary source verification tracking for licenses and create token-authenticated external API endpoints for system integration.

## Purpose
Primary source verification lets users confirm license validity against state board websites and track verification history — critical for compliance audits. External API enables other systems (workforce tools, project management) to query license and compliance data programmatically.

## Output
- License verification recording workflow with state board lookup links
- Verification history on license detail page
- `api_tokens` table with token-based auth middleware
- External API at `/api/v1/` with key read-only endpoints
- CLI command to manage API tokens
</objective>

  <name>Task 1: License Verification Tracking</name>
  <files>licenses/schema.sql, licenses/migrations.py, licenses/db.py, api/licenses.py, frontend/templates/licenses/license_detail.html</files>
  <action>
    **Schema migration (licenses/migrations.py):**
    - Add `last_verified_date TEXT` and `verification_status TEXT CHECK(verification_status IN ('unverified','verified','not_found','discrepancy'))` columns to `state_licenses`
    - Default verification_status to 'unverified'
    - ALTER TABLE approach (two ADD COLUMN statements)
    - Add 'verified' to license_events event_type CHECK constraint — use migration to recreate constraint (SQLite limitation: create new table, copy data, rename) OR simply insert verification events as a separate `license_verifications` table
    - DECISION: Use a new `license_verifications` table instead of modifying the CHECK constraint on license_events. Simpler migration, cleaner separation.

    **New table `license_verifications` (licenses/schema.sql):**
    ```sql
    CREATE TABLE IF NOT EXISTS license_verifications (
        id              TEXT PRIMARY KEY,
        license_id      TEXT NOT NULL,
        verified_date   TEXT NOT NULL,
        status          TEXT NOT NULL CHECK (status IN ('verified','not_found','discrepancy')),
        source          TEXT DEFAULT 'manual',  -- 'manual' or future 'automated'
        board_url       TEXT,                    -- URL used for verification
        notes           TEXT,
        created_by      TEXT DEFAULT 'system',
        created_at      TEXT NOT NULL DEFAULT (datetime('now')),
        FOREIGN KEY (license_id) REFERENCES state_licenses(id) ON DELETE CASCADE
    );
    CREATE INDEX IF NOT EXISTS idx_verifications_license ON license_verifications(license_id);
    ```

    **DB functions (licenses/db.py):**
    - `record_verification(conn, license_id, status, notes, board_url, created_by)` — inserts into license_verifications, updates state_licenses.last_verified_date and verification_status
    - `get_verification_history(conn, license_id)` — returns verification records ordered by verified_date DESC
    - `get_board_lookup_url(conn, state_code)` — returns the lookup_url from state_license_boards (may already exist via get_license_board)

    **API routes (api/licenses.py):**
    - POST `/api/licenses/<license_id>/verify` — accepts {status, notes}, records verification, returns updated license
    - GET `/api/licenses/<license_id>/verifications` — returns verification history

    **UI (frontend/templates/licenses/license_detail.html):**

    - Add "Last verified: <date>" text if verified
    - Add "Verify License" button in the action area
    - Verify modal: shows board lookup URL as clickable link (opens new tab), radio buttons for result (verified/not_found/discrepancy), notes textarea, submit button
    - On submit, POST to verify API, refresh the detail section

    **Avoid:**
    - Do NOT modify the license_events CHECK constraint (SQLite table rebuild is risky)
    - Do NOT attempt automated web scraping (future feature)

<task type="auto">
  <name>Task 2: External API with Token Authentication</name>
  <files>auth/schema.sql, auth/db.py, auth/migrations.py, api/external.py, api/__init__.py, cli/main.py</files>
  <action>
    **Schema (auth/schema.sql + auth/migrations.py):**
    ```sql
    CREATE TABLE IF NOT EXISTS api_tokens (
        id          TEXT PRIMARY KEY,
        name        TEXT NOT NULL,           -- descriptive name ("workforce-sync")
        token_hash  TEXT NOT NULL UNIQUE,     -- SHA-256 hash of the token
        permissions TEXT DEFAULT 'read',      -- 'read' or 'read,write'
        created_by  TEXT,
        created_at  TEXT NOT NULL DEFAULT (datetime('now')),
        expires_at  TEXT,                     -- NULL = no expiry
        last_used   TEXT,
        is_active   INTEGER DEFAULT 1
    );
    ```
    - Migration adds table via CREATE TABLE IF NOT EXISTS

    **Auth functions (auth/db.py):**
    - `create_api_token(conn, name, permissions, created_by, expires_days)` — generates random token, stores SHA-256 hash, returns plaintext token (shown once)
    - `validate_api_token(conn, token)` — hashes token, looks up in api_tokens, checks is_active and expires_at, updates last_used, returns token record or None
    - `list_api_tokens(conn)` — returns all tokens (without hash)
    - `revoke_api_token(conn, token_id)` — sets is_active=0

    **External API blueprint (api/external.py):**
    - New Flask blueprint `external_bp` mounted at `/api/v1`
    - `@require_api_token` decorator: checks X-API-Key header, validates via auth/db, returns 401 if invalid
    - Endpoints (all read-only, all require valid token):
      - GET `/api/v1/licenses` — paginated list (page, per_page params), returns {items, total, page, per_page, pages}
      - GET `/api/v1/licenses/<id>` — full license detail with status, events, scopes
      - GET `/api/v1/compliance/summary` — {total, active, expiring_30, expiring_60, expired, compliance_rate}
      - GET `/api/v1/employees` — list employees with license counts
      - GET `/api/v1/health` — {status: "ok", version: "0.2.0"} (no auth required)

    **Register blueprint (api/__init__.py):**
    - Import and register external_bp in create_app()
    - No session/login required (token auth is separate)

    **CLI command (cli/main.py or licenses CLI):**
    - `qms licenses api-token create --name "workforce-sync"` — creates token, prints plaintext once
    - `qms licenses api-token list` — shows active tokens (name, created, last_used, permissions)
    - `qms licenses api-token revoke <token_id>` — deactivates token

    **Avoid:**
    - Do NOT store plaintext tokens (hash with SHA-256)
    - Do NOT allow write operations in v1 (read-only for safety)
    - Do NOT add rate limiting beyond what already exists
  </action>

## Goal
Complete Vital schedule extraction — process the remaining 54 schedule sheets using a two-pass approach: Docling (IBM TableFormer) for structured table extraction, then Claude vision agents to fill coverage gaps. Produce a complete equipment manifest from schedule data.

## Purpose
Plan 25-02 validated the harness with 5 sheets (203 entries) using pdfplumber agents. Testing revealed:
- **PyMuPDF find_tables()**: Fast (0.1s/page) but produces jumbled, unusable output on CAD-generated schedule drawings
- **Docling**: Slower (47s/page) but produces clean structured DataFrames with proper column mapping and merged cell handling
- **Claude vision**: Excellent contextual understanding, catches tables Docling misses

The optimal approach: Docling extracts structured tables (high accuracy, proper columns), Claude vision agents handle sheets where Docling misses tables or has low coverage. Both run locally/via subscription — zero additional cost.

Remaining 54 sheets: Architectural (34), Electrical (10), Utility (3), Mechanical (3), Structural (3), Refrigeration (1).

  <files>pipeline/docling_extractor.py</files>
  <action>
    **Create `pipeline/docling_extractor.py`:**

    A thin wrapper around Docling's DocumentConverter that extracts schedule tables from a PDF and returns harness-compatible equipment dicts.

    ```python
    def extract_tables_from_pdf(file_path: str) -> list[dict]:
        """Extract all tables from a PDF using Docling.

        Returns list of dicts, each with:
          - table_index: int
          - headers: list of column names
          - rows: list of dicts (column_name → value)
          - row_count: int
          - col_count: int
        """
    ```

    ```python
    def tables_to_equipment(tables: list[dict], discipline: str) -> list[dict]:
        """Convert Docling table output to harness-compatible equipment entries.

        Maps common column patterns to schedule_extractions fields:
          - MARK/TAG → tag
          - AREA SERVED/DESCRIPTION → description
          - HP/HP EACH → hp
          - V/PH/HZ/VOLTAGE → voltage
          - MANUFACTURER → manufacturer
          - MODEL → model_number
          - WEIGHT/WEIGHT (LBS) → weight_lbs
          - CFM → cfm
          - KW/KVA → kva

        Returns list of equipment dicts ready for store_schedule_data().
        """
    ```

    Key design:
    - Initialize DocumentConverter once at module level (reuse across calls)
    - Column name matching is case-insensitive and fuzzy (e.g., "WEIGHT (LBS)" matches weight_lbs)
    - Skip rows where the tag/mark column is empty
    - For architectural schedules: map door/window/finish-specific columns
    - Return empty list if no tables found (not an error)

    Avoid:
    - Do NOT modify extraction_harness.py or schedule_extractor.py
    - Do NOT add CLI commands — this is called from the Claude Code session
    - Keep the Docling dependency optional (import inside function, graceful fallback message)
  </action>
  <verify>

  <action>
    **This is an execution task.** Process all 54 remaining sheets using a two-pass approach:

    **Pass 1 — Docling extraction (batch all sheets):**
    For each pending sheet:
    1. Run extract_tables_from_pdf(file_path)
    2. Convert via tables_to_equipment(tables, discipline)
    3. If entries found: record via harness (record_result)
    4. If no entries: flag sheet for Pass 2

    Process in batches of 5, save checkpoint after each batch.

    **Pass 2 — Claude vision agents (gap-fill):**
    For sheets where Docling found zero tables or suspiciously few entries:
    1. Launch a Claude Code agent to read the PDF
    2. Agent extracts equipment using visual understanding of the schedule tables
    3. Record results via harness

    **Batch order (by discipline priority):**
    - Refrigeration: 1 sheet
    - Utility: 3 sheets
    - Mechanical: 3 sheets
    - Electrical: 10 sheets
    - Structural: 3 sheets
    - Architectural: 34 sheets (doors, windows, finishes)

    **Architectural mapping:**
    - Door schedules → equipment_type "Door", tag from MARK column
    - Window schedules → equipment_type "Window"
    - Finish schedules → equipment_type "Room Finish" with room number as tag
    - If a sheet has no extractable schedule tables, record empty result

    **Empty sheets**: Some schedule drawings have calculation tables or notes rather than equipment schedules. Record with empty entries — do not fabricate.

    **Error handling**: If extraction fails, call h.record_error(). Harness retries once, then skips.

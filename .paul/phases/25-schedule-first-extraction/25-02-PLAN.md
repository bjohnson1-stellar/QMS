<objective>
## Goal
Build the extraction harness — a session-aware orchestrator that dispatches schedule sheets to Claude Code agents, tracks progress in the database, pauses before hitting session limits, and resumes from checkpoints in new sessions. Then run it on Vital's 59 schedule sheets to validate.

## Purpose
Phase 25-01 built the data management layer (what to extract, where to store). This plan builds the **orchestration layer** that actually runs the extraction pipeline. Without the harness, processing 59 sheets requires manual per-sheet orchestration. With it, extraction is a single command that runs autonomously, pauses gracefully, and resumes across sessions.

Key harness requirements:
- **Session-aware**: tracks context pressure, pauses before limits, saves state
- **Checkpoint-based**: every sheet stored atomically, resume from last checkpoint
- **Agent-dispatched**: spawns Claude Code agents to read PDFs (no API calls)
- **Progress tracking**: per-sheet status in DB, summary reporting
- **Error handling**: retry once on failure, then skip + flag for review

## Output
- `pipeline/extraction_harness.py` — Session-aware extraction orchestrator
- Vital schedule extraction: 59 schedule sheets processed, data in staging table
</objective>

    **Create `pipeline/extraction_harness.py`:**

    The harness is the orchestration layer that Claude Code (the session) calls to manage extraction. It does NOT spawn sub-agents itself — instead, it provides a **sheet-at-a-time** interface that the Claude Code session drives.

    **Core design principle**: The harness is a state machine that the Claude Code session steps through. Each step = one sheet. The session reads the PDF, extracts data, and feeds it back to the harness. This keeps the AI work inside the Claude Code session (user's subscription) while the harness handles all the bookkeeping.

    **`ExtractionHarness` class:**

    ```python
    class ExtractionHarness:
        def __init__(self, project_id, phase="schedules"):
            self.project_id = project_id
            self.phase = phase
            self.batch_size = 5
            self.state_file = Path(".paul/extraction-state.json")

        def get_status(self) -> dict:
            """Current extraction progress."""
            # Returns: total, completed, pending, current_batch, errors

        def next_sheet(self) -> Optional[dict]:
            """Get the next sheet to process. Returns None if phase complete."""
            # Returns sheet info with file_path for the agent to read

        def record_result(self, sheet_id, entries, status="success"):
            """Record extraction result for a sheet."""
            # Calls store_schedule_data() and updates state

        def record_error(self, sheet_id, error_msg):
            """Record a failed extraction."""
            # Logs error, marks sheet for retry or skip

        def save_checkpoint(self):
            """Save current state for session resumption."""
            # Writes to .paul/extraction-state.json

        def is_batch_complete(self) -> bool:
            """True if current batch of N sheets is done."""

        def get_batch_summary(self) -> str:
            """Human-readable summary of current batch progress."""
    ```

    **State file** (`.paul/extraction-state.json`):
    ```json
    {
        "project_id": 7,
        "phase": "schedules",
        "started_at": "2026-03-26T12:00:00",
        "last_sheet_id": 1715,
        "last_sheet_number": "U6002",
        "sheets_completed": 20,
        "sheets_total": 59,
        "equipment_found": 145,
        "errors": [],
        "paused_at": "2026-03-26T12:30:00"
    }
    ```

    **`get_status()`** checks:
    1. Read schedule_extractions for completed sheet count
    2. Read extraction_order for total schedule sheets
    3. Compute pending = total - completed
    4. Load state file for error history and timing

    **`next_sheet()`**:
    1. Call get_pending_schedules() for unprocessed sheets
    2. Return first pending sheet (dict with id, drawing_number, discipline, file_path)
    3. Return None if all sheets processed

    **`record_result(sheet_id, entries, status)`**:
    1. Call store_schedule_data(sheet_id, project_id, entries)
    2. Update state file with latest progress
    3. Increment completed count

    **`record_error(sheet_id, error_msg)`**:
    1. Log error with sheet details
    2. Add to errors list in state
    3. If first error for this sheet, mark for retry
    4. If second error, skip and flag

    **Convenience functions** (module-level):

    `start_extraction(project_id, phase="schedules")` → Creates harness, returns status.

    `resume_extraction(project_id)` → Loads state file, creates harness at checkpoint, returns status.

    `get_extraction_prompt(sheet)` → Returns the formatted prompt text for the Claude Code session to use when reading the PDF. This is the bridge between the harness and the agent — the harness generates the prompt, the session reads the PDF and applies it.

    Avoid:
    - Do NOT call anthropic API — the harness provides prompts, the Claude Code session does the AI work
    - Do NOT spawn sub-processes or agents from Python — the Claude Code session orchestrates
    - Do NOT make the harness async — it's a synchronous state machine stepped by the session
    - Keep state file small — progress details live in the database, state file is just a pointer

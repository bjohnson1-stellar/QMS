<objective>
## Goal
Build the floor plan extraction data layer and extractor module — `floor_plan_extractions` table for staging results, `floor_plan_extractor.py` with context-aware vision prompts that use the schedule-built equipment checklist, and harness integration for session-stepped batch extraction.

## Purpose
Phase 25-26 established 452 schedule entries and 661 equipment instances, but all data comes from schedules (tabular). Floor plans, P&IDs, and diagrams show WHERE equipment is physically located, how it's connected, and spatial relationships — data that schedules can't provide. The 283 non-schedule MEP sheets at Vital are sitting unextracted. This plan builds the infrastructure; Plan 27-02 will run the first batch (Refrigeration P&IDs).

The key innovation: prompts inject the schedule-built equipment checklist so the AI asks "which of these 452 known tags appear here?" instead of "what do you see?" — preventing hallucination (per feedback from PP1114 incident).

## Output
- `floor_plan_extractions` table added to equipment_schema.sql
- `pipeline/floor_plan_extractor.py` module with store/query/prompt functions
- Harness adapted to support `phase="plans"` with floor plan storage
- Validated with 1 test extraction on a Refrigeration P&ID
</objective>

    ### A. Add floor_plan_extractions table to equipment_schema.sql

    Add AFTER the schedule_extractions table definition:

    ```sql
    CREATE TABLE IF NOT EXISTS floor_plan_extractions (
        id INTEGER PRIMARY KEY,
        sheet_id INTEGER NOT NULL REFERENCES sheets(id),
        project_id INTEGER NOT NULL REFERENCES projects(id),
        tag TEXT NOT NULL,
        location_area TEXT,
        location_room TEXT,
        grid_reference TEXT,
        appearance_type TEXT DEFAULT 'physically_shown'
            CHECK(appearance_type IN ('physically_shown', 'referenced', 'legend')),
        description TEXT,
        equipment_type TEXT,
        confidence REAL DEFAULT 0.85,
        extraction_model TEXT,
        page_number INTEGER,
        additional_attributes TEXT,
        created_at TEXT DEFAULT CURRENT_TIMESTAMP,
        UNIQUE(sheet_id, tag)
    );

    CREATE INDEX IF NOT EXISTS idx_floor_plan_extractions_project
        ON floor_plan_extractions(project_id, tag);
    ```

  <files>pipeline/floor_plan_extractor.py, pipeline/extraction_harness.py</files>

    ### A. Add prompt builder to floor_plan_extractor.py

    Create `build_floor_plan_prompt(project_id, sheet_id, discipline)`:
    - Call `context_builder.get_context_for_extraction(project_id, sheet_id)` to get equipment checklist
    - Build a discipline-specific vision prompt:

    ```
    You are analyzing a {discipline} drawing ({drawing_number}).

    KNOWN EQUIPMENT FOR THIS PROJECT (from schedules):
    {equipment_checklist — filtered to same discipline first, then cross-discipline}

    TASK: Examine this drawing and identify which of the known equipment tags
    are PHYSICALLY SHOWN on this drawing (not just referenced in notes/legends).

    For each equipment tag found:
    - tag: exact tag as shown on drawing
    - location_area: area/zone name if visible
    - location_room: room number/name if visible
    - grid_reference: column/row grid reference if visible (e.g., "C-4")
    - appearance_type: "physically_shown" if the equipment is drawn/depicted,
      "referenced" if only mentioned in a note/callout, "legend" if in legend only
    - confidence: 0.0-1.0 how confident you are this tag appears

    IMPORTANT: Only report equipment you can actually see depicted or labeled.
    Do NOT fabricate entries from the checklist. If a tag from the checklist is
    NOT on this drawing, do not include it. It is better to miss equipment than
    to report equipment that isn't shown.

    Also report any equipment tags you see that are NOT on the checklist — these
    may be unlisted equipment needing review.

    Return as JSON array of objects.
    ```

    - Add discipline-specific notes:
      - Refrigeration P&IDs: "Look for RCU, RAHU, valves, piping connections, instrument tags"
      - Electrical plans: "Look for panel tags, disconnect tags, motor connections, conduit routing"
      - Mechanical plans: "Look for AHU, VAV, FPB, ductwork routing, equipment tags in title blocks"
      - Utility plans: "Look for boilers, pumps, tanks, piping, equipment tags"
      - Plumbing plans: "Look for fixture tags (WC, LAV, FD), pipe routing, equipment schedules"

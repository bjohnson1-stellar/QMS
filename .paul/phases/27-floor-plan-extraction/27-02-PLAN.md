<objective>
## Goal
Extract equipment from 19 Refrigeration sheets using the calibrate-first-then-scale approach: run 3 calibration sheets through the full four-phase pipeline (Sonnet discovery → Haiku detail → Sonnet conflict → Opus shadow), refine prompts based on error analysis, then scale to the remaining 16 sheets with proven prompts.

## Purpose
This is the first time the floor plan extraction pipeline runs on real drawings. Systematic prompt errors multiply at scale — 1 bad pattern × 100 sheets = 100 bad extractions. Running 3 calibration sheets first (~15 min) catches these patterns before they propagate. Refrigeration is the pilot discipline (most complex, highest value).

## Output
- floor_plan_extractions populated with calibrated Refrigeration equipment data
- shadow_reviews, extraction_misses, accuracy_log populated with QA metrics
- gold_standard entries for 1-2 user-verified sheets
- Refined prompts proven on pilot batch, ready for 27-03 (remaining MEP)
</objective>

<task type="auto">
  <name>Task 2: Calibration Batch — 100% Opus Shadow + Phase 3 Haiku Detail</name>
  <files>pipeline/floor_plan_extractor.py</files>
  <action>
    **Execution task. No code changes.**

    **Step A — Opus shadow review on all 3 calibration sheets:**

    For each sheet, dispatch Opus agent (BLIND — no access to Phase 2 results):
    ```
    Agent(
      model="opus",
      prompt="Perform independent blind extraction of equipment from this drawing.
        File: {file_path}
        Drawing: {drawing_number}
        This is a Refrigeration drawing.

        Read the PDF. Identify ALL equipment tags visible.
        For each: tag, location_area, location_room, grid_reference,
          appearance_type (physically_shown/referenced/legend),
          description, equipment_type, confidence

        You have NO prior extraction results. Fresh, blind pass.
        Return ONLY a JSON array."
    )
    ```

    Compare each Opus result against Phase 2 Sonnet result:
    - Confirmed (both found, match)
    - Discrepancy (both found, differ)
    - Over-extraction (Sonnet found, Opus didn't — potential hallucination)
    - Under-extraction (Opus found, Sonnet didn't — missed tag)

    Record in shadow_reviews table:
    ```sql
    INSERT INTO shadow_reviews (sheet_id, extraction_model, review_model,
      items_checked, items_correct, items_missed, items_wrong, accuracy, notes)
    VALUES (?, 'claude-sonnet-vision', 'opus', ?, ?, ?, ?, ?, ?);
    ```

    Record misses in extraction_misses:
    ```sql
    INSERT INTO extraction_misses (sheet_id, extraction_model, found_by,
      missed_item_type, missed_item, severity)
    VALUES (?, 'claude-sonnet-vision', 'opus-shadow', 'equipment', ?, ?);
    ```

    **Step B — Phase 3 Haiku detail on confirmed tags:**

    For each calibration sheet with Phase 2 tags:
    1. Query Phase 2 results to get confirmed tag list for this sheet
    2. Dispatch Haiku agent with ONLY those tags:
       ```
       Agent(
         model="haiku",
         prompt="Targeted detail extraction on Refrigeration drawing.
           File: {file_path}
           These specific tags were confirmed on this drawing:
           {tag_list_from_phase_2}

           For EACH tag, find: location_area, location_room, grid_reference,
           connections, additional visible attributes.
           Search ONLY for these tags — do not add new ones.
           Return JSON array."
       )
       ```
    3. UPDATE existing floor_plan_extractions rows with location detail
  </action>

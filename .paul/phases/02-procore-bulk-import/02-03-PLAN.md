<objective>
## Goal
Add attachment URL tracking during CSV import and vector indexing of quality issue descriptions, completing the Phase 2 bulk import pipeline.

## Purpose
Attachments close the loop on data fidelity — Procore observations include photo URLs that must be captured so photos can be retrieved later (Phase 4 mobile pipeline or manual download). Vector indexing enables semantic search across all imported quality issues, which Phase 3 dashboard needs for cross-project pattern discovery.

## Output
- Import engine captures attachment/photo URLs from CSV into `quality_issue_attachments`
- New `index_quality_issues()` function in vectordb indexer
- `quality_issues` collection in ChromaDB with title+description+location text
- CLI `qms quality index` command to trigger indexing
- Tests covering both features
</objective>

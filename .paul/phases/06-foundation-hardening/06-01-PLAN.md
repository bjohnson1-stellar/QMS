<objective>
## Goal
Eliminate N+1 query patterns in the licenses module and add audit trail logging to all license CRUD operations.

## Purpose
The licenses module currently makes per-license subqueries for scopes and CE credits inside loops — creating N+1 patterns that will degrade as license count grows. Additionally, no CRUD operations are audit-logged, meaning changes are untraceable. Both issues must be fixed before building new features on top.

## Output
- `licenses/db.py` — batch-loading functions, audit trail helper, N+1 fixes in `get_ce_compliance_report()`
- `api/licenses.py` — batch scope loading in state detail and CSV export routes, audit calls on all mutations
</objective>

<task type="auto">
  <name>Task 2: Add audit trail to all license CRUD operations</name>
  <files>licenses/db.py, api/licenses.py</files>
  <action>
    1. In licenses/db.py, add helper `_audit(conn, entity_type, entity_id, action, old_values=None, new_values=None, changed_by='system')`:
       - Inserts into `audit_log` table using the existing core schema
       - `old_values` and `new_values` serialized as JSON strings via `json.dumps()`
       - `entity_type` will be one of: 'license', 'ce_credit', 'ce_requirement'

    2. Add audit calls to existing CRUD functions in licenses/db.py:
       - `create_license()`: audit with action='created', new_values=data dict
       - `update_license()`: fetch old values BEFORE update, audit with action='updated', old+new values
       - `delete_license()`: fetch record BEFORE delete, audit with action='deleted', old_values=record
       - `create_ce_credit()`: audit with action='created', entity_type='ce_credit'
       - `update_ce_credit()`: fetch old, audit with action='updated'
       - `delete_ce_credit()`: fetch old, audit with action='deleted'
       - `create_ce_requirement()`: audit with action='created', entity_type='ce_requirement'
       - `update_ce_requirement()`: fetch old, audit with action='updated'
       - `delete_ce_requirement()`: audit with action='deleted'

    3. In api/licenses.py, pass `changed_by` from session:
       - For each mutation endpoint, extract `session.get("user", {}).get("email", "system")`
       - Pass as `created_by` or `changed_by` parameter to db functions
       - The db functions will forward this to `_audit()`

    Avoid: Adding audit to read-only operations. Do NOT change the audit_log schema in core/schema.sql. Use the existing table as-is.
  </action>

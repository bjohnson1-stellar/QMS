Create the license notification backend: schema tables, rule-driven notification generation, and CLI command — cloning the proven welding notification pattern.

- `qms licenses check-notifications` CLI command

<task type="auto">
  <name>Task 2: Create notification generation engine</name>
  <files>licenses/notifications.py</files>
  <action>
    Create `licenses/notifications.py` by adapting `welding/notifications.py` pattern:

    **Core functions (clone from welding):**
    - `get_notification_rules(conn)` → fetch active rules
    - `create_notification(conn, ...)` → insert with dedup via UNIQUE constraint
    - `acknowledge_notification(conn, notification_id, user)` → status → acknowledged
    - `resolve_notification(conn, notification_id, user)` → status → resolved
    - `cleanup_old_notifications(conn, days=90)` → delete old resolved
    - `update_days_until_due(conn)` → refresh days_until_due using JULIANDAY
    - `list_active_notifications(conn, limit=50)` → sorted by priority, due_date
    - `get_notification_summary(conn)` → by_status, by_priority, by_type, urgent items

    **License-specific generators:**

    `generate_expiration_notifications(conn, rules)`:
    - Filter rules where notification_type = 'expiration_warning'
    - Query state_licenses WHERE status = 'active' AND expiration_date IS NOT NULL
      AND expiration_date BETWEEN DATE('now') AND DATE('now', '+N days')
    - Join with holder_name, state_code, license_number for message
    - Title format: "License Expiring: {holder_name} ({state_code}) #{license_number}"
    - Message includes: holder name, license type, state, expiration date, days remaining

    `generate_ce_deadline_notifications(conn, rules)`:
    - Filter rules where notification_type = 'ce_deadline'
    - Query ce_requirements JOIN state_licenses
      WHERE ce period end date is approaching (within days_before)
    - Check if credits_earned < credits_required for the period
    - Title format: "CE Deadline: {holder_name} ({state_code}) - {credits_needed} hrs needed"
    - Only notify if credits are actually incomplete

    `generate_renewal_reminder_notifications(conn, rules)`:
    - Filter rules where notification_type = 'renewal_reminder'
    - Query state_licenses WHERE status = 'renewal_pending'
      OR (status = 'active' AND expiration_date within days_before)
    - Title format: "Renewal Needed: {holder_name} ({state_code}) #{license_number}"
    - Message includes action needed: submit renewal application

    `generate_all_notifications(conn)`:
    - Get rules, call all three generators, update_days_until_due
    - Return stats dict: {expiration: {created, skipped}, ce: {...}, renewal: {...}, total: {...}}

    **Key differences from welding pattern:**
    - entity_id is TEXT (UUID) not INTEGER
    - License queries use state_licenses table (not weld_wpq)
    - CE deadline logic queries ce_requirements + ce_credits tables
    - No continuity concept — replaced by renewal_reminder type
  </action>

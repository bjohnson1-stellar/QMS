<acceptance_criteria>

## AC-1: Upload Modal
```gherkin
Given the M4 tab is active
When user clicks "Upload SOP" button in the M4 header area
Then a modal appears with file drop zone and file picker
And only PDF files are accepted
And uploading shows a progress/spinner state
And on success, the intake queue view opens showing the new intake
```

## AC-2: Intake Queue View
```gherkin
Given user navigates to the intake queue (via "Intake Queue" button on M4 tab)
When the queue loads
Then all intake records display in a table/list with: file name, status badge, suggested category, date
And status filter tabs (All / Pending / Classified / Approved / Rejected) filter the list
And clicking an intake row opens the classification review panel
```

## AC-3: Classification Review & Approve/Reject
```gherkin
Given a classified intake is selected
When the review panel loads
Then AI suggestions display: category, scope tags, programs, code references, summary, suggested title/doc_id
And user can edit title and document_id before approving
And "Approve" button POSTs to /api/intakes/{id}/approve with overrides → shows success, refreshes queue
And "Reject" button prompts for optional reason, POSTs to /api/intakes/{id}/reject → shows success
And non-classified intakes (pending, analyzing, error) show appropriate status messaging
```

## AC-4: SOP Lifecycle Actions on Detail View
```gherkin
Given a SOP detail view is open (existing renderSopDetail function)
When the SOP status is "draft"
Then an "Approve" button appears that calls POST /api/sops/{id}/approve
When the SOP status is "approved"
Then a "Publish" button appears that calls POST /api/sops/{id}/publish
And after action, the detail view refreshes to show new status
```

Build the SOP intake backend pipeline: file upload endpoint, AI-powered PDF classification via Claude API, and intake management endpoints for reviewing/approving classified SOPs.

- Extended `qualitydocs/db.py` — Helper functions for intake-to-SOP conversion

<acceptance_criteria>

## AC-1: PDF Upload & Storage
```gherkin
Given a user uploads a PDF file via POST /qualitydocs/api/sops/upload
When the file is a valid PDF (≤50MB)
Then the file is saved to data/quality-documents/intake/{intake_id}/{sanitized_filename}
And a qm_sop_intake record is created with status='pending' and file_hash (SHA-256)
And the response returns the intake record with id
```

## AC-2: AI Classification
```gherkin
Given an intake record exists with status='pending'
When the classification function is called (triggered by upload or manually)
Then Claude API receives the PDF (base64) with a structured prompt
And the response populates: suggested_category_id, suggested_scope_tags, suggested_program_ids, suggested_document_id, ai_classification (full JSON)
And intake status changes to 'classified' (or 'error' on failure)
```

## AC-3: Intake Management Endpoints
```gherkin
Given classified intake records exist
When GET /qualitydocs/api/intakes is called (optional ?status= filter)
Then paginated intake list is returned with intake details + suggested metadata

Given a classified intake record
When POST /qualitydocs/api/intakes/{id}/approve is called with optional user_overrides
Then a new qm_sops record is created in 'draft' status using AI suggestions (or overrides)
And qm_sop_code_references are created from AI-detected codes
And qm_program_sops linkages are created from suggested program_ids
And intake status changes to 'approved' with final_sop_id set
And qm_sop_history records the 'created' action

Given a classified intake record
When POST /qualitydocs/api/intakes/{id}/reject is called with optional reason
Then intake status changes to 'rejected' with error_message set to reason
```

<task type="auto">
  <name>Task 2: Claude-powered SOP classification</name>
  <files>qualitydocs/classifier.py, qualitydocs/__init__.py</files>
  <action>
    Create qualitydocs/classifier.py with classify_sop(intake_id) function:

    1. Load intake record, read PDF file as base64
    2. Call Claude API (Sonnet model) using same pattern as pipeline/extractor.py:
       - import anthropic lazily
       - Model mapping dict (haiku/sonnet/opus)
       - Send PDF as base64 document (application/pdf media type)
    3. Structured prompt asks Claude to return JSON with:
       - suggested_category: {code, name, confidence} — match to one of 15 M4 categories
       - suggested_scope_tags: string[] — relevant scope tags
       - suggested_programs: [{id, name, confidence}] — match to M3 programs (SIS-3.01 through SIS-3.05)
       - code_references: [{code, organization, section, original_text}] — detected industry codes
       - summary: string — 2-3 sentence description of the SOP
       - suggested_document_id: string — e.g., "SOP-004-012" based on category code
    4. Prompt includes the 15 category definitions and 5 program definitions as context
    5. Parse JSON response (handle markdown code fences in response)
    6. Look up category by code → get category_id
    7. Look up programs by name → get program_ids
    8. Generate next document_id: query max existing document_id for the category, increment
    9. Update intake record:
       - ai_classification = full JSON response
       - suggested_category_id = resolved category_id
       - suggested_scope_tags = scope_tags array
       - suggested_program_ids = program_ids array
       - suggested_document_id = generated document_id
       - status = 'classified'
       - processed_at = now
    10. On error: set status='error', error_message=str(e)

    Add to qualitydocs/__init__.py exports: classify_sop

    Avoid:
    - Don't use vision/image mode — use document (PDF) media type for better text extraction
    - Don't hardcode API key — use ANTHROPIC_API_KEY env var via anthropic.Anthropic()
    - Don't fail silently — log errors and set intake status='error'
  </action>
  <verify>
    python -c "from qualitydocs.classifier import classify_sop; print('import ok')"
    → Module imports without error
    → Function signature accepts intake_id parameter
    Manual test: Upload a real SOP PDF → intake record updated with classification data
  </verify>
  <done>AC-2 satisfied: AI classification populates intake with category, scope, programs, codes, summary</done>
</task>

<task type="auto">
  <name>Task 3: Intake management endpoints (list, detail, approve, reject)</name>
  <files>api/qualitydocs.py, qualitydocs/db.py</files>
  <action>
    Add endpoints to api/qualitydocs.py:

    1. GET /qualitydocs/api/intakes
       - Query params: status (optional filter), page, per_page
       - Returns paginated list using standard {items, total, page, per_page, pages} pattern
       - Each item includes: id, file_name, status, suggested_category (joined name), suggested_document_id, created_at, processed_at

    2. GET /qualitydocs/api/intakes/{id}
       - Returns full intake record with:
         - Parsed ai_classification JSON
         - Parsed suggested_scope_tags, suggested_program_ids
         - Category name (joined from qm_categories)
         - Program names (joined from qm_programs)

    3. POST /qualitydocs/api/intakes/{id}/approve
       - Body (optional): {category_id, scope_tags, program_ids, document_id, title, summary} — user overrides
       - Merge overrides with AI suggestions (overrides win)
       - Create SOP via create_sop() with status='draft':
         - document_id, title (from file_name if not provided), category_id, scope_tags, file_path, file_hash, content_text (from ai_classification), summary, version='1.0', revision='A'
       - Create qm_sop_code_references from ai_classification.code_references
       - Create qm_program_sops links from program_ids
       - Record qm_sop_history: action='created', actor='system', notes='Created from intake #{id}'
       - Update intake: status='approved', final_sop_id=new_sop_id, user_overrides=overrides
       - Return the new SOP record

    4. POST /qualitydocs/api/intakes/{id}/reject
       - Body: {reason} (optional)
       - Update intake: status='rejected', error_message=reason
       - Return updated intake

    Add helpers in db.py:
    - approve_intake(id, overrides) — Wraps the approve logic in a single transaction
    - create_code_references(sop_id, references_list) — Bulk insert code references
    - next_document_id(category_code) — Generate next SOP-{cat_code}-{NNN} number
    - list_intakes_paginated(status, page, per_page) — Paginated version of list_intakes

    Avoid:
    - Don't allow approving non-classified intakes (must be status='classified')
    - Don't allow duplicate approvals (check status before proceeding)
  </action>

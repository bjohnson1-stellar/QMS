<acceptance_criteria>

## AC-1: Entity List Page
```gherkin
Given the user navigates to /licenses/entities
When the page loads
Then they see entity stats cards (total, active, registrations count)
And a searchable/filterable table of business entities
And each row shows name, type, status, license count, registration count
And clicking an entity navigates to its detail page
```

## AC-2: Entity CRUD from List Page
```gherkin
Given an editor/admin is on the entity list page
When they click "+ Add Entity"
Then a modal appears with fields: name, entity_type, status, ein, state_of_formation, parent_id
When they submit valid data
Then the entity appears in the list without page reload
And edit/delete actions work from the entity detail page
```

## AC-3: Entity Detail Page
```gherkin
Given the user navigates to /licenses/entities/<id>
When the page loads
Then they see entity details (name, type, EIN, state, status, parent link)
And a registrations section with add/edit/delete capability
And a linked licenses section showing all licenses with entity_id = this entity
And a child entities section (if any subsidiaries/DBAs exist)
```

## AC-4: Registration Management
```gherkin
Given an editor is on the entity detail page
When they click "+ Add Registration"
Then a modal appears with fields: registration_type, state_code, registration_number, status, filed_date, expiration_date, notes
When they submit valid data
Then the registration appears in the list
And edit/delete work via inline actions
```

## AC-5: License-Entity Linking
```gherkin
Given an editor is on the entity detail page
When they view the linked licenses section
Then they see all licenses currently linked to this entity
And each license links to its detail page
And a "Link License" action allows assigning unlinked licenses to this entity
```

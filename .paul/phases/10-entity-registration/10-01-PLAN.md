<objective>
## Goal
Create the backend foundation for business entity and registration tracking — schema, CRUD operations, migration to link existing licenses, and API endpoints.

## Purpose
SIS manages professional and business licenses across multiple states. Currently `business_entity` is just a text field on `state_licenses`. This plan creates proper entity records with parent/subsidiary hierarchy and registration tracking (Secretary of State filings, DBE certifications, minority certs), enabling centralized entity management.

## Output
- `business_entities` and `entity_registrations` tables with hierarchy support
- DB functions for entity CRUD, registration CRUD, and entity-license linking
- API endpoints for full entity and registration management
- Migration to add `entity_id` FK on `state_licenses` and auto-link existing licenses
</objective>

<task type="auto">
  <name>Task 1: Add entity schema and migrate existing data</name>
  <files>licenses/schema.sql, licenses/migrations.py</files>
  <action>
    **Schema additions to licenses/schema.sql:**

    1. Create `business_entities` table:
       - id TEXT PRIMARY KEY (UUID)
       - name TEXT NOT NULL (company name)
       - entity_type TEXT NOT NULL CHECK IN ('corporation','llc','partnership','sole_proprietorship','subsidiary','dba','branch') DEFAULT 'corporation'
       - parent_id TEXT (FK to business_entities.id for hierarchy — NULL = top-level)
       - ein TEXT (Employer Identification Number — optional)
       - state_of_incorporation TEXT (2-letter code)
       - address TEXT, city TEXT, state_code TEXT, zip_code TEXT
       - phone TEXT, website TEXT, notes TEXT
       - status TEXT NOT NULL DEFAULT 'active' CHECK IN ('active','inactive','dissolved')
       - created_at, updated_at, created_by (standard fields)
       - Indexes on: parent_id, entity_type, status, state_of_incorporation

    2. Create `entity_registrations` table:
       - id TEXT PRIMARY KEY (UUID)
       - entity_id TEXT NOT NULL FK to business_entities.id ON DELETE CASCADE
       - registration_type TEXT NOT NULL CHECK IN ('secretary_of_state','dbe','mbe','wbe','sbe','hub','sdvosb','other')
       - state_code TEXT NOT NULL (2-letter)
       - registration_number TEXT
       - issuing_authority TEXT (e.g. "Florida Department of State")
       - issued_date TEXT, expiration_date TEXT
       - status TEXT NOT NULL DEFAULT 'active' CHECK IN ('active','expired','pending','revoked','suspended')
       - filing_frequency TEXT CHECK IN ('annual','biennial','triennial','one_time',NULL)
       - next_filing_date TEXT (calculated or manually set)
       - fee_amount REAL
       - notes TEXT
       - created_at, updated_at, created_by
       - Indexes on: entity_id, registration_type, state_code, status, expiration_date
       - UNIQUE(entity_id, registration_type, state_code)

    **Migration in licenses/migrations.py:**

    3. Add migration function `_migrate_entity_tables(conn)`:
       - CREATE TABLE IF NOT EXISTS for both tables
       - ALTER TABLE state_licenses ADD COLUMN entity_id TEXT (with try/except for already-exists)
       - CREATE INDEX on state_licenses(entity_id)
       - Auto-link: SELECT DISTINCT business_entity FROM state_licenses WHERE business_entity IS NOT NULL
       - For each unique business_entity name: INSERT INTO business_entities (id, name, entity_type) and UPDATE state_licenses SET entity_id = ? WHERE business_entity = ?
       - Register in MIGRATION_VERSION tracking

    Avoid: Do NOT drop or alter the existing business_entity TEXT column — keep it for backward compatibility. The entity_id FK is the canonical link going forward.
  </action>
  <verify>
    Run: python -c "from licenses.migrations import migrate; migrate()"
    Then: sqlite3 query to verify business_entities table exists, entity_registrations table exists, state_licenses.entity_id column exists
    Check: SELECT count(*) FROM business_entities should match SELECT count(DISTINCT business_entity) FROM state_licenses WHERE business_entity IS NOT NULL
  </verify>
  <done>AC-1 satisfied: Both tables created with all columns and constraints. AC-4 satisfied: Existing licenses auto-linked to entity records.</done>
</task>

  <name>Task 2: Add entity and registration DB functions</name>
  <files>licenses/db.py, licenses/__init__.py</files>
  <action>
    **Add to licenses/db.py — Entity functions:**

    1. `list_entities(search=None, entity_type=None, status='active', parent_id=None, page=1, per_page=25)`:
       - Paginated query with optional filters
       - Include child_count (COUNT of entities where parent_id = this.id)
       - Include license_count (COUNT of state_licenses where entity_id = this.id)
       - Include registration_count (COUNT of entity_registrations where entity_id = this.id)
       - Return {items, total, page, per_page, pages} dict (matches pagination pattern from Phase 6)

    2. `get_entity(entity_id)`:
       - Single entity with all fields
       - Include children (list of direct child entities)
       - Include registrations (list from entity_registrations)
       - Include licenses (list from state_licenses WHERE entity_id = ?)
       - Include days_until_expiry for each registration

    3. `create_entity(name, entity_type='corporation', parent_id=None, **kwargs)`:
       - UUID generation, audit trail via _audit()
       - Validate parent_id exists if provided
       - Return new entity id

    4. `update_entity(entity_id, **kwargs)`:
       - Partial update pattern, audit trail
       - Return success boolean

    5. `delete_entity(entity_id)`:
       - Check for linked licenses (block if any exist, or unlink)
       - CASCADE deletes registrations
       - Audit trail

    6. `get_entity_hierarchy(entity_id=None)`:
       - If entity_id: return entity + all descendants (recursive CTE)
       - If None: return all top-level entities with children nested
       - Useful for tree display

    **Add to licenses/db.py — Registration functions:**

    7. `list_registrations(entity_id=None, registration_type=None, state_code=None, status=None, expiring_days=None)`:
       - Filter by entity, type, state, status
       - expiring_days: registrations expiring within N days
       - Include entity_name via JOIN

    8. `get_registration(registration_id)`:
       - Single registration with entity_name

    9. `create_registration(entity_id, registration_type, state_code, **kwargs)`:
       - UUID, audit trail
       - Validate entity_id exists

    10. `update_registration(registration_id, **kwargs)`:
        - Partial update, audit trail

    11. `delete_registration(registration_id)`:
        - Delete with audit trail

    **Update licenses/__init__.py:**
    - Export new public functions

    Follow existing patterns: use `_get_conn()`, `_audit()`, UUID generation with `str(uuid.uuid4())`, standard error handling.

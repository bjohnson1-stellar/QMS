<task type="auto">
  <name>Task 2: Employee credential portfolio page</name>
  <files>frontend/templates/licenses/credential_portfolio.html, api/licenses.py, licenses/db.py</files>
  <action>
    Create employee credential portfolio at /licenses/credentials:

    **DB function (licenses/db.py):**
    - Add `get_employee_portfolio(conn, employee_id)` — returns all licenses for an employee with CE summary per license:
      ```sql
      SELECT sl.*, cr_summary.hours_earned, ce_req.hours_required, ce_req.period_months
      FROM state_licenses sl
      LEFT JOIN (SELECT license_id, SUM(hours) AS hours_earned FROM ce_credits WHERE status='approved' GROUP BY license_id) cr_summary ON cr_summary.license_id = sl.id
      LEFT JOIN ce_requirements ce_req ON ce_req.state_code = sl.state_code AND ce_req.license_type = sl.license_type
      WHERE sl.employee_id = ?
      ORDER BY sl.expiration_date ASC
      ```
    - Add `list_employees_with_licenses(conn)` — returns employees who have at least one license, with license count and compliance summary:
      ```sql
      SELECT e.id, e.first_name, e.last_name, e.employee_number,
             COUNT(sl.id) AS license_count,
             SUM(CASE WHEN sl.status='active' THEN 1 ELSE 0 END) AS active_count,
             SUM(CASE WHEN sl.status='expired' THEN 1 ELSE 0 END) AS expired_count
      FROM employees e
      JOIN state_licenses sl ON sl.employee_id = e.id
      GROUP BY e.id
      ORDER BY e.last_name, e.first_name
      ```

    **API route (api/licenses.py):**
    - Add page route `/credentials` rendering `licenses/credential_portfolio.html`
    - Add API route `/api/credentials/employees` returning list_employees_with_licenses
    - Add API route `/api/credentials/<employee_id>` returning get_employee_portfolio

    **Template (credential_portfolio.html):**
    - Extends base.html, breadcrumb: Licenses / Credentials
    - **Employee list** (left or full-width): Table of employees with license count, active/expired badges, compliance indicator (green check if all CE met, yellow warning if behind, red if expired licenses)
    - **Employee detail panel** (shown on click or separate view):
      - Header: Employee name + number
      - Summary card: Total licenses, active count, CE compliance %
      - License cards: Each license shows type, state, status badge, expiration, CE progress bar (hours earned / required)
      - CE progress bars reuse the pattern from license_detail.html (line 296-309)

    **Pattern:** Master-detail layout. Click employee row → fetch portfolio via API → render detail section. Follow existing inline style patterns.

<objective>
## Goal
Create the CE provider and course catalog backend — new schema tables, DB CRUD functions, API endpoints, and cross-state credit applicability mapping. Migrate existing free-text provider/course fields to linkable catalog entries.

## Purpose
Currently CE credits store provider and course as free-text strings, making it impossible to track providers, manage course catalogs, or map credits across states. This plan creates the structured catalog that enables provider management, course browsing, and cross-state CE credit applicability — the "Harbor Compliance" CE management equivalent.

## Output
- 3 new database tables: `ce_providers`, `ce_courses`, `ce_credit_courses`
- DB CRUD functions for providers and courses
- API endpoints for provider and course management
- Cross-state applicability via `states_accepted` JSON on courses
- Backward-compatible: existing free-text provider/course_name fields remain functional
</objective>

## AC-1: CE Providers CRUD
```gherkin
Given the ce_providers table exists with columns (id, name, accreditation_body, accreditation_number, contact_email, contact_phone, website, notes, is_active, created_at, updated_at)
When a user calls POST /api/ce-providers with {name, accreditation_body}
Then a provider record is created and returned with 201
And audit_log records the creation

When GET /api/ce-providers is called
Then all active providers are returned (filterable by ?active=true/false, ?search=term)

When PUT /api/ce-providers/<id> is called with updated fields
Then the provider is updated and audit-logged

When DELETE /api/ce-providers/<id> is called by admin
Then the provider is soft-deleted (is_active=false) and audit-logged
```

## AC-2: CE Courses CRUD with Cross-State Mapping
```gherkin
Given the ce_courses table exists with columns (id, provider_id FK, title, description, hours, format, states_accepted JSON, license_types JSON, url, is_active, created_at, updated_at)
When a user calls POST /api/ce-courses with {provider_id, title, hours, states_accepted: ["FL","TX","GA"]}
Then a course record is created linking to the provider
And the states_accepted field stores which states accept this course for CE credit

When GET /api/ce-courses is called with ?state_code=FL
Then only courses where states_accepted includes "FL" are returned

When GET /api/ce-courses is called with ?provider_id=<id>
Then only courses from that provider are returned
```

## AC-3: Credit-Course Linking
```gherkin
Given the ce_credit_courses junction table exists (credit_id FK, course_id FK)
When a CE credit is created with an optional course_id parameter
Then a link is created in ce_credit_courses
And the credit's course_name and provider fields can auto-populate from the course catalog

When GET /api/ce-credits is called with existing course links
Then each credit includes course and provider details in the response
```

  <name>Task 1: Create CE provider and course schema + migrations</name>
  <files>licenses/schema.sql, licenses/migrations.py</files>
  <action>
    Add three new tables to licenses/schema.sql:

    1. `ce_providers` table:
       - id TEXT PRIMARY KEY (UUID)
       - name TEXT NOT NULL UNIQUE
       - accreditation_body TEXT
       - accreditation_number TEXT
       - contact_email TEXT
       - contact_phone TEXT
       - website TEXT
       - notes TEXT
       - is_active INTEGER DEFAULT 1
       - created_at TEXT DEFAULT (datetime('now'))
       - updated_at TEXT DEFAULT (datetime('now'))

    2. `ce_courses` table:
       - id TEXT PRIMARY KEY (UUID)
       - provider_id TEXT REFERENCES ce_providers(id)
       - title TEXT NOT NULL
       - description TEXT
       - hours REAL NOT NULL
       - format TEXT CHECK IN ('online', 'classroom', 'self-study', 'webinar', 'conference', 'other')
       - states_accepted TEXT DEFAULT '[]' (JSON array of state codes)
       - license_types TEXT DEFAULT '[]' (JSON array of license types this course satisfies)
       - url TEXT
       - is_active INTEGER DEFAULT 1
       - created_at TEXT DEFAULT (datetime('now'))
       - updated_at TEXT DEFAULT (datetime('now'))
       - UNIQUE(provider_id, title)

    3. `ce_credit_courses` junction table:
       - credit_id TEXT NOT NULL REFERENCES ce_credits(id) ON DELETE CASCADE
       - course_id TEXT NOT NULL REFERENCES ce_courses(id) ON DELETE CASCADE
       - PRIMARY KEY (credit_id, course_id)

    Add migration function in licenses/migrations.py:
    - `_migrate_ce_catalog()` — CREATE TABLE IF NOT EXISTS for all 3 tables
    - Add indexes: ce_courses(provider_id), ce_courses(is_active), ce_providers(is_active)
    - Register in the migrations list

    Do NOT alter existing ce_credits or ce_requirements tables — keep backward compatible.

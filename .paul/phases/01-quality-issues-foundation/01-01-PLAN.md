<objective>
## Goal
Create the quality issues module with a unified schema for all quality issue types (observations, NCRs, CARs, deficiencies, punch items), supporting tables (attachments, audit history, issue links, tags, corrective actions, root causes), normalization config, and proper FK relationships to existing QMS entities.

## Purpose
This is the foundation for unified quality intelligence across all Procore projects. Every subsequent phase (import, dashboard, mobile capture, push) builds on this schema. The audit trail, issue links, and tagging system are architectural investments that compound — they cannot be effectively retrofitted later.

## Output
- New `quality/` module directory with `schema.sql`, `__init__.py`, `db.py`
- 8 new tables in quality.db
- `core/db.py` updated with `quality` in SCHEMA_ORDER
- `config.yaml` updated with normalization mappings and quality config
</objective>

<task type="auto">
  <name>Task 1: Create quality module with schema</name>
  <files>quality/__init__.py, quality/schema.sql, quality/db.py</files>
  <action>
    Create `quality/` directory at package root (sibling to projects/, welding/, etc.).

    **quality/__init__.py** — empty module init.

    **quality/schema.sql** — Create 8 tables following QMS conventions (INTEGER PRIMARY KEY, TEXT timestamps, CREATE TABLE IF NOT EXISTS):

    1. `root_causes` — lookup table
       - id, name (UNIQUE), description, category, is_active DEFAULT 1, created_at
       - INSERT OR IGNORE seed data: Workmanship, Materials, Design/Engineering, Environmental, Procedural, Subcontractor, Equipment/Tools, Other

    2. `tags` — flexible tag definitions
       - id, name (UNIQUE), color, description, created_at

    3. `quality_issues` — main table
       - id INTEGER PRIMARY KEY
       - type TEXT NOT NULL CHECK (type IN ('observation', 'ncr', 'car', 'deficiency', 'punch', 'other'))
       - title TEXT NOT NULL
       - description TEXT
       - project_id INTEGER REFERENCES projects(id)
       - business_unit_id INTEGER REFERENCES business_units(id)
       - location TEXT (freetext — building/area/room)
       - trade TEXT (normalized trade name)
       - severity TEXT CHECK (severity IN ('low', 'medium', 'high', 'critical'))
       - priority TEXT CHECK (priority IN ('low', 'medium', 'high', 'urgent'))
       - status TEXT DEFAULT 'open' CHECK (status IN ('open', 'in_review', 'in_progress', 'resolved', 'closed', 'deferred'))
       - root_cause_id INTEGER REFERENCES root_causes(id)
       - assigned_to TEXT REFERENCES employees(id)
       - reported_by TEXT
       - due_date TEXT
       - resolved_by TEXT REFERENCES employees(id)
       - resolved_at TEXT
       - resolution_notes TEXT
       - estimated_cost REAL
       - source TEXT DEFAULT 'manual' CHECK (source IN ('manual', 'procore', 'mobile', 'import'))
       - source_id TEXT (original ID from external system for dedup)
       - source_project_id TEXT (Procore project ID)
       - source_url TEXT (link back to Procore observation)
       - source_synced_at TEXT (last sync timestamp)
       - metadata TEXT (JSON for type-specific fields)
       - created_at TEXT DEFAULT CURRENT_TIMESTAMP
       - updated_at TEXT DEFAULT CURRENT_TIMESTAMP
       - Indexes: project_id, business_unit_id, type, status, source+source_id (UNIQUE for dedup), trade, severity

    4. `quality_issue_attachments`
       - id, issue_id REFERENCES quality_issues(id) ON DELETE CASCADE
       - filename TEXT NOT NULL, filepath TEXT NOT NULL
       - file_type TEXT (image, document, video, audio)
       - file_size INTEGER
       - description TEXT
       - source_url TEXT (original URL from Procore)
       - created_at TEXT DEFAULT CURRENT_TIMESTAMP
       - Index on issue_id

    5. `quality_issue_history` — audit trail
       - id, issue_id REFERENCES quality_issues(id) ON DELETE CASCADE
       - field_changed TEXT NOT NULL
       - old_value TEXT, new_value TEXT
       - changed_by TEXT REFERENCES employees(id)
       - changed_at TEXT DEFAULT CURRENT_TIMESTAMP
       - Index on issue_id, changed_at

    6. `quality_issue_links` — relationships
       - id, issue_id_a REFERENCES quality_issues(id) ON DELETE CASCADE
       - issue_id_b REFERENCES quality_issues(id) ON DELETE CASCADE
       - link_type TEXT NOT NULL CHECK (link_type IN ('related', 'duplicate', 'caused_by', 'parent_child'))
       - notes TEXT
       - created_at TEXT DEFAULT CURRENT_TIMESTAMP
       - UNIQUE(issue_id_a, issue_id_b, link_type)

    7. `quality_issue_tags` — junction table
       - id, issue_id REFERENCES quality_issues(id) ON DELETE CASCADE
       - tag_id INTEGER REFERENCES tags(id) ON DELETE CASCADE
       - created_at TEXT DEFAULT CURRENT_TIMESTAMP
       - UNIQUE(issue_id, tag_id)

    8. `corrective_actions`
       - id, issue_id REFERENCES quality_issues(id) ON DELETE CASCADE
       - action_type TEXT DEFAULT 'corrective' CHECK (action_type IN ('corrective', 'preventive', 'containment'))
       - description TEXT NOT NULL
       - assigned_to TEXT REFERENCES employees(id)
       - status TEXT DEFAULT 'open' CHECK (status IN ('open', 'in_progress', 'completed', 'verified', 'ineffective'))
       - due_date TEXT
       - completed_at TEXT
       - verified_by TEXT REFERENCES employees(id)
       - verified_at TEXT
       - effectiveness_notes TEXT
       - created_at TEXT DEFAULT CURRENT_TIMESTAMP
       - updated_at TEXT DEFAULT CURRENT_TIMESTAMP
       - Index on issue_id, status

    **quality/db.py** — Helper functions:
    - `get_root_causes(conn)` — return all active root causes
    - `get_tags(conn)` — return all tags
    - `normalize_trade(trade_name: str) -> str` — apply normalization from config
    - `normalize_status(status: str) -> str` — apply normalization from config
    - `log_issue_change(conn, issue_id, field, old_val, new_val, changed_by)` — insert audit trail row
  </action>
  <verify>
    Run: python -c "from qms.quality import db; print('quality module imports OK')"
    Verify schema.sql parses: python -c "import sqlite3; conn=sqlite3.connect(':memory:'); conn.executescript(open('quality/schema.sql').read()); print([r[0] for r in conn.execute(\"SELECT name FROM sqlite_master WHERE type='table'\").fetchall()])"
  </verify>
  <done>AC-1 satisfied: 8 tables created. AC-5 satisfied: root_causes seeded.</done>
</task>

<objective>
## Goal
Extend the qualitydocs schema with new tables for M3 Quality Programs and M4 SOPs, including categories, scope tags, program-SOP linkage, and draft/approval lifecycle. Create a `qualitydocs/db.py` module with all DB access functions for the new tables.

## Purpose
The existing schema supports M1/M2 (modules → sections → subsections → content blocks) but has no structure for M3 programs or M4 SOPs with categories and scopes. This plan builds the data foundation that all subsequent UI phases depend on.

## Output
- Extended `qualitydocs/schema.sql` with ~8 new tables
- New `qualitydocs/db.py` with CRUD-ready DB functions
- Migration-safe: existing M1/M2 data untouched
</objective>

  <name>Task 1: Extend qualitydocs/schema.sql with M3/M4 tables</name>
  <files>qualitydocs/schema.sql</files>
  <action>
    Add the following tables to the END of schema.sql (after existing tables, preserving all current DDL):

    1. **qm_programs** — M3 Quality Programs
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - program_id TEXT UNIQUE NOT NULL (e.g., "SIS-3.01")
       - title TEXT NOT NULL
       - description TEXT
       - primary_codes TEXT (JSON array of governing codes, e.g., '["ASME BPVC IX", "AWS D1.1"]')
       - qualification_requirements TEXT (prose or JSON)
       - status TEXT CHECK(status IN ('draft','under_review','approved','published','superseded','obsolete')) DEFAULT 'draft'
       - version TEXT DEFAULT '1.0'
       - created_at TEXT DEFAULT (datetime('now'))
       - updated_at TEXT DEFAULT (datetime('now'))

    2. **qm_categories** — M4 SOP Categories
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - category_code TEXT UNIQUE NOT NULL (e.g., "4.01")
       - module_number INTEGER DEFAULT 4
       - name TEXT NOT NULL
       - description TEXT
       - parent_program_id INTEGER REFERENCES qm_programs(id) ON DELETE SET NULL
       - display_order INTEGER
       - created_at TEXT DEFAULT (datetime('now'))

    3. **qm_sops** — M4 Standard Operating Procedures
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - document_id TEXT UNIQUE NOT NULL (e.g., "SIS-4.04.015")
       - title TEXT NOT NULL
       - category_id INTEGER NOT NULL REFERENCES qm_categories(id)
       - status TEXT CHECK(status IN ('draft','under_review','approved','published','superseded','obsolete')) DEFAULT 'draft'
       - scope_tags TEXT DEFAULT '[]' (JSON array of scope strings)
       - version TEXT DEFAULT '1.0'
       - revision TEXT DEFAULT 'A'
       - file_path TEXT (path to original PDF)
       - file_hash TEXT
       - content_text TEXT (extracted text from PDF for search)
       - summary TEXT (AI-generated summary)
       - approved_by TEXT
       - approved_at TEXT
       - effective_date TEXT
       - review_cycle_months INTEGER DEFAULT 24
       - created_at TEXT DEFAULT (datetime('now'))
       - updated_at TEXT DEFAULT (datetime('now'))

    4. **qm_program_sops** — M3-M4 linkage (many-to-many)
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - program_id INTEGER NOT NULL REFERENCES qm_programs(id) ON DELETE CASCADE
       - sop_id INTEGER NOT NULL REFERENCES qm_sops(id) ON DELETE CASCADE
       - UNIQUE(program_id, sop_id)

    5. **qm_sop_code_references** — Code references detected in SOPs
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - sop_id INTEGER NOT NULL REFERENCES qm_sops(id) ON DELETE CASCADE
       - code TEXT NOT NULL
       - organization TEXT
       - code_section TEXT
       - original_text TEXT
       - detection_method TEXT CHECK(detection_method IN ('explicit','ai_detected','prose_detected'))

    6. **qm_sop_intake** — Intake/classification pipeline tracking
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - file_name TEXT NOT NULL
       - file_path TEXT
       - file_hash TEXT
       - status TEXT CHECK(status IN ('pending','analyzing','classified','approved','rejected','error')) DEFAULT 'pending'
       - ai_classification TEXT (JSON blob with Claude's full recommendation)
       - suggested_category_id INTEGER REFERENCES qm_categories(id)
       - suggested_scope_tags TEXT (JSON array)
       - suggested_program_ids TEXT (JSON array of program IDs)
       - suggested_document_id TEXT
       - user_overrides TEXT (JSON blob of any manual adjustments)
       - final_sop_id INTEGER REFERENCES qm_sops(id)
       - error_message TEXT
       - created_at TEXT DEFAULT (datetime('now'))
       - processed_at TEXT

    7. **qm_sop_history** — SOP revision/approval history
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - sop_id INTEGER NOT NULL REFERENCES qm_sops(id) ON DELETE CASCADE
       - action TEXT NOT NULL CHECK(action IN ('created','submitted','approved','rejected','published','revised','superseded'))
       - actor TEXT
       - notes TEXT
       - previous_status TEXT
       - new_status TEXT
       - created_at TEXT DEFAULT (datetime('now'))

    Also extend qm_content_fts to include SOP content by adding an update trigger or plan for Phase 18 to rebuild FTS. For now, just add a comment noting the FTS extension is planned.

    Do NOT modify any existing table DDL. Only append new CREATE TABLE statements.
  </action>
  <verify>

import sqlite3, pathlib
sql = pathlib.Path('qualitydocs/schema.sql').read_text()

for t in ['qm_programs','qm_categories','qm_sops','qm_program_sops','qm_sop_code_references','qm_sop_intake','qm_sop_history']:
    assert t in tables, f'Missing table: {t}'
# Verify existing tables still present
for t in ['qm_modules','qm_sections','qm_subsections','qm_content_blocks']:
    assert t in tables, f'Existing table missing: {t}'
print('All tables verified')
"

  <files>qualitydocs/db.py, qualitydocs/__init__.py</files>
  <action>
    Create qualitydocs/db.py with the following functions, following existing QMS patterns (use get_db() from core, return dicts via sqlite3.Row):

    **Programs (M3):**
    - list_programs(status=None) → list of program dicts
    - get_program(program_id: str) → program dict or None (by program_id like "SIS-3.01")
    - get_program_by_id(id: int) → program dict or None
    - create_program(program_id, title, description=None, primary_codes=None, status='draft') → int (new id)
    - update_program(id, **kwargs) → bool
    - get_program_sops(program_id: str) → list of SOP dicts linked to this program

    **Categories (M4):**
    - list_categories(module_number=4) → list of category dicts with sop_count
    - get_category(category_code: str) → category dict or None
    - create_category(category_code, name, description=None, parent_program_id=None, display_order=None) → int
    - seed_categories() → seeds the 15 M4 categories from the Category Roadmap if not already present

    **SOPs (M4):**
    - list_sops(category_id=None, scope=None, status=None, page=1, per_page=50) → paginated dict {items, total, page, per_page, pages}
    - get_sop(document_id: str) → full SOP dict with category info and program links
    - get_sop_by_id(id: int) → SOP dict or None
    - create_sop(document_id, title, category_id, scope_tags=None, file_path=None, file_hash=None, content_text=None, summary=None) → int (new id, status='draft')
    - update_sop(id, **kwargs) → bool (sets updated_at)
    - approve_sop(id, approved_by) → bool (sets status='approved', approved_by, approved_at)
    - publish_sop(id) → bool (sets status='published')
    - link_sop_to_program(sop_id, program_id) → bool
    - search_sops(query, limit=50) → list of matching SOP dicts (LIKE search on title + content_text)

    **SOP History:**
    - add_sop_history(sop_id, action, actor=None, notes=None, previous_status=None, new_status=None) → int
    - get_sop_history(sop_id) → list of history dicts ordered by created_at DESC

    **Intake:**
    - create_intake(file_name, file_path=None, file_hash=None) → int (new id, status='pending')
    - update_intake(id, **kwargs) → bool
    - get_intake(id) → intake dict or None
    - list_intakes(status=None) → list of intake dicts

    **Implementation notes:**
    - Use `import json` for scope_tags and other JSON fields (json.loads on read, json.dumps on write)
    - Follow pagination pattern: {items, total, page, per_page, pages} matching v0.2 convention
    - approve_sop and publish_sop should also call add_sop_history internally
    - create_sop should also call add_sop_history with action='created'
    - For list_sops scope filter: use json_each(scope_tags) if SQLite supports it, otherwise LIKE '%"refrigeration"%' pattern
    - All functions use get_db() connection from core

    Update qualitydocs/__init__.py to export the key public functions from db.py.
  </action>
  <verify>

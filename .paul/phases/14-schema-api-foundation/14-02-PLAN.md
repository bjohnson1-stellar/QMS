<acceptance_criteria>

## AC-1: Programs API
```gherkin
Given the qualitydocs blueprint is registered
When GET /qualitydocs/api/programs is called
Then it returns a JSON array of all programs
And GET /qualitydocs/api/programs/<program_id> returns a single program with linked SOPs
```

## AC-2: Categories API
```gherkin
Given M4 categories are seeded
When GET /qualitydocs/api/categories is called
Then it returns a JSON array of categories with sop_count
And GET /qualitydocs/api/categories/<code> returns a single category
```

## AC-3: SOPs List and Detail API
```gherkin
Given SOPs exist in the database
When GET /qualitydocs/api/sops is called with optional ?category_id=&status=&scope=&page=&per_page= params
Then it returns paginated results {items, total, page, per_page, pages}
And GET /qualitydocs/api/sops/<document_id> returns full SOP detail with programs and parsed scope_tags
```

## AC-4: SOP Lifecycle API
```gherkin
Given a SOP exists in draft status
When POST /qualitydocs/api/sops/<id>/approve is called with {approved_by}
Then the SOP status changes to approved and history is recorded
And POST /qualitydocs/api/sops/<id>/publish changes status to published
```

## AC-5: SOP Search API
```gherkin
Given SOPs exist with titles and content
When GET /qualitydocs/api/sops/search?q=... is called
Then it returns matching SOPs by title or content_text
```

    Add new endpoints to the existing blueprint (below the existing M1/M2 routes):

    **Programs (M3):**
    - GET /api/programs — list_programs(status=request.args.get("status"))
    - GET /api/programs/<program_id> — get_program(program_id) + get_program_sops(program_id)

    **Categories (M4):**
    - GET /api/categories — list_categories(module_number=request.args.get("module", 4, type=int))
    - GET /api/categories/<code> — get_category(code)

    **SOPs (M4):**
    - GET /api/sops — list_sops with query params (category_id, status, scope, page, per_page)
    - GET /api/sops/search?q= — search_sops(q, limit)
    - GET /api/sops/<document_id> — get_sop(document_id)
    - POST /api/sops — create_sop (JSON body: document_id, title, category_id, scope_tags, summary)
    - POST /api/sops/<int:id>/approve — approve_sop(id, approved_by from JSON)
    - POST /api/sops/<int:id>/publish — publish_sop(id)
    - POST /api/sops/<int:id>/link-program — link_sop_to_program(sop_id, program_id from JSON)
    - GET /api/sops/<int:id>/history — get_sop_history(id)

    **Import from db.py:**
    Add imports for: list_programs, get_program, get_program_sops, list_categories, get_category,
    list_sops, get_sop, create_sop, approve_sop, publish_sop, link_sop_to_program, search_sops,
    get_sop_history

    **Pattern:** Follow existing endpoint style — jsonify responses, 404 for missing resources, input validation at API boundary.

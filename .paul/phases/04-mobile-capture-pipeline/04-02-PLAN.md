<objective>
## Goal
Extend the mobile capture pipeline with voice note transcription (OpenAI Whisper API) and a web review page for captured observations. Audio files are transcribed to text, analyzed by Claude for structured extraction, and created as quality issues — completing the full phone-to-database pipeline.

## Purpose
Brandon records voice notes alongside photos on site. Voice notes describe issues hands-free while walking the job. The review page lets him verify and correct AI analysis before the observations get pushed to Procore (Phase 5).

## Output
- `quality/mobile_capture.py` — extended with audio transcription + analysis
- `api/quality.py` — new `/captures` route + `/api/captures` JSON endpoint
- `frontend/templates/quality/captures.html` — review page template
- `quality/cli.py` — capture command updated to handle audio
- `config.yaml` — added audio extensions and whisper model config
- `pyproject.toml` — `openai` added to pipeline extras
- `tests/test_mobile_capture.py` — extended with voice + review tests
</objective>

## AC-1: Audio File Discovery
```gherkin
Given a capture folder containing .m4a, .mp3, .wav, and .ogg audio files
When the capture processor scans the folder
Then it discovers audio files alongside image files
And it skips files already processed (tracked in capture_log)
```

## AC-2: Voice Note Transcription
```gherkin
Given an audio file from the capture folder
When the file is sent to OpenAI Whisper API
Then the spoken content is transcribed to text
And the transcript is returned for analysis
And API errors are handled gracefully (returns error dict)
```

## AC-3: Transcript Analysis
```gherkin
Given a transcribed voice note text
When the text is sent to Claude with the structured extraction prompt
Then the response contains: title, type, trade, severity, location, description
And the response is parsed into a dict matching quality_issues schema fields
And the original transcript is preserved in metadata
```

    1. Add module-level optional import for openai:
       ```python
       try:
           import openai as openai_sdk
       except ImportError:
           openai_sdk = None
       ```

    2. Add audio constants:
       ```python
       _AUDIO_EXTENSIONS = {".m4a", ".mp3", ".wav", ".ogg"}
       _ALL_EXTENSIONS = _IMAGE_EXTENSIONS | _AUDIO_EXTENSIONS
       ```

    3. Update `scan_capture_folder()`:
       - Change filter from `_IMAGE_EXTENSIONS` to `_ALL_EXTENSIONS`
       - No other changes needed (dedup logic stays the same)

    4. Add `transcribe_audio(audio_path: Path, model: str = "whisper-1") -> Dict`:
       - Check openai_sdk is not None, return error dict if missing
       - Call `openai_sdk.OpenAI().audio.transcriptions.create(model=model, file=f)`
       - Return `{"text": transcript.text}` on success
       - Return `{"error": str(e)}` on failure
       - Log transcription result

    5. Add `analyze_transcript(text: str, model: str = "sonnet") -> Dict`:
       - Similar to analyze_photo but sends text instead of image
       - System prompt adapted for voice note analysis:
         "Analyze this voice note transcript from a construction site visit..."
       - Same structured JSON output: title, type, trade, severity, location, description, recommended_action
       - Same normalization (normalize_trade, type validation)
       - Preserve original transcript in result dict as "transcript" key

    6. Update `process_captures()`:
       - After scanning, split files into images and audio by extension
       - For images: existing analyze_photo flow
       - For audio: transcribe_audio → if success → analyze_transcript → create_issue
       - Audio issues get file_type='audio' in quality_issue_attachments
       - Audio issues store transcript in metadata JSON
       - Summary dict adds "transcribed" count

    **Update config.yaml mobile_capture section:**
    - Add audio extensions to supported_extensions list
    - Add whisper_model: "whisper-1" setting

    **Update pyproject.toml:**
    - Add `openai>=1.0.0` to the `pipeline` extras group (where anthropic already lives)

    **Avoid:**
    - Do NOT use async — keep synchronous
    - Do NOT convert audio formats — Whisper API handles m4a/mp3/wav/ogg natively
    - Do NOT change existing photo analysis flow
    - Do NOT make openai a hard dependency — module-level try/except like anthropic

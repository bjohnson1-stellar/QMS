Add Documents, Notes, and Activity Feed UI sections to the license detail page, wired to the 09-01 backend APIs.

## AC-1: Document Upload UI
```gherkin
Given a license detail page with editor role
When user clicks "Upload Documents", selects files, picks a doc type, and submits
Then files are uploaded via POST /licenses/{id}/documents and appear in the documents list
```

## AC-2: Document List and Download
```gherkin
Given a license has uploaded documents
When the detail page loads
Then documents are listed with filename, type badge, size, upload date, and a download link
And clicking download fetches GET /documents/{id}/download
```

## AC-3: Document Delete (Admin)
```gherkin
Given an admin viewing a license with documents
When admin clicks delete on a document
Then DELETE /documents/{id} is called and the document is removed from the list
```

## AC-4: Notes Section
```gherkin
Given a license detail page with editor role
When user types a note and clicks "Add Note"
Then POST /licenses/{id}/notes is called and the note appears in the notes list
And notes show text, author, timestamp, with admin-only delete
```

## AC-5: Activity Feed Timeline
```gherkin
Given a license with events, notes, and documents
When the detail page loads
Then the Activity section shows a unified timeline from GET /licenses/{id}/activity
And each entry shows type icon/badge, timestamp, and relevant details
```

<task type="auto">
  <name>Task 1: Add Documents, Notes, and Activity Feed sections to license detail page</name>
  <files>frontend/templates/licenses/license_detail.html</files>
  <action>
    Add three new sections to `license_detail.html` between the Event History section and the CE Credits section. Follow the existing card + inline style patterns already in the template.

    **1. Documents Section** (card):
    - Header: "Documents" with "+ Upload" button (editor only)
    - Document list loaded via `fetch(API + '/licenses/' + LIC_ID + '/documents')` on page load
    - Each doc row: type badge (colored by doc type), original_filename as download link, file size (human-readable), uploaded_at date, uploaded_by
    - Admin-only delete button per doc
    - Empty state: "No documents attached."

    **2. Notes Section** (card):
    - Header: "Notes" with inline add form (editor only): textarea + "Add Note" button
    - Notes list loaded via `fetch(API + '/licenses/' + LIC_ID + '/notes')` on page load
    - Each note: text content, created_by, created_at timestamp
    - Admin-only delete button per note
    - Empty state: "No notes."

    **3. Activity Feed Section** (card):
    - Header: "Activity"
    - Feed loaded via `fetch(API + '/licenses/' + LIC_ID + '/activity?limit=50')` on page load
    - Each entry: activity_type badge (event=blue, note=gray, document=green), timestamp, details
      - event: event_type + event_date + notes
      - note: note text (truncated to ~200 chars)
      - document: "Uploaded {filename} ({doc_type})"
    - Reverse chronological (API returns this way)
    - Empty state: "No activity recorded."

    **Upload Modal** for documents:
    - File input with `multiple` attribute
    - Doc type dropdown: certificate, application, correspondence, receipt, bond, insurance, other
    - Submit sends FormData to `POST /licenses/{id}/documents` with `files` + `doc_type`
    - On success, refresh the documents list and activity feed (no full page reload — use JS refresh)

    **JavaScript functions** (add to existing `<script>` block):
    - `loadDocuments()` — fetch + render document list into `#docList`
    - `loadNotes()` — fetch + render notes into `#noteList`
    - `loadActivity()` — fetch + render activity feed into `#activityFeed`
    - `uploadDocuments(e)` — FormData upload handler
    - `addNote(e)` — POST note, refresh notes + activity
    - `deleteDoc(id)` — DELETE with confirm, refresh docs + activity
    - `deleteNote(id)` — DELETE with confirm, refresh notes + activity
    - Call `loadDocuments()`, `loadNotes()`, `loadActivity()` on page load (add to IIFE or DOMContentLoaded)

    **Avoid:**
    - Do NOT use full page reload for document/note operations — use fetch + DOM update for snappy UX
    - Do NOT add new CSS files — use inline styles matching existing patterns
    - Do NOT change existing sections (Details, Scopes, Portal Creds, Events, CE Credits, modals)
    - Keep the existing `const API`, `LIC_ID`, etc. variable declarations unchanged
  </action>

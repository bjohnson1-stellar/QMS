<objective>
## Goal
Add license_documents and license_notes tables with full CRUD operations and API routes, plus a unified activity feed query that merges events, notes, and document uploads into a single timeline.

## Purpose
Licenses need attached documents (certificates, applications, bonds, insurance) and timestamped notes for compliance tracking. The activity feed gives a complete audit trail of all license lifecycle actions in one view.

## Output
- 2 new schema tables (license_documents, license_notes)
- DB functions for document CRUD, note CRUD, and unified activity feed
- API routes for document upload/download/delete, note CRUD, and activity feed
- File storage at `data/license-documents/{license_id}/`
</objective>

## AC-1: Document Storage
```gherkin
Given a license exists in the system
When a user uploads a document (PDF, image, or office file) with a document type
Then the file is saved to data/license-documents/{license_id}/{filename}
And a license_documents record is created with type, filename, original_filename, file_size, mime_type, uploaded_by, uploaded_at
And the document can be downloaded via GET /licenses/api/documents/{doc_id}/download
```

## AC-2: Document CRUD
```gherkin
Given a license has attached documents
When a user lists documents for that license via GET /licenses/api/licenses/{id}/documents
Then all documents are returned sorted by uploaded_at descending
And documents can be deleted via DELETE /licenses/api/documents/{doc_id} (admin only)
```

## AC-3: Notes CRUD
```gherkin
Given a license exists
When a user creates a note via POST /licenses/api/licenses/{id}/notes with text content
Then a license_notes record is created with note_text, created_by, created_at
And notes can be listed via GET /licenses/api/licenses/{id}/notes (sorted newest first)
And notes can be deleted via DELETE /licenses/api/notes/{note_id} (admin only)
```

## AC-4: Unified Activity Feed
```gherkin
Given a license has events, notes, and documents
When the activity feed is requested via GET /licenses/api/licenses/{id}/activity
Then all three types are merged into a single list sorted by timestamp descending
And each item has: type (event|note|document), timestamp, description, created_by
And the feed supports optional ?limit parameter (default 50)
```

## AC-5: Bulk Document Upload
```gherkin
Given a license exists
When a user uploads multiple files in a single POST request to /licenses/api/licenses/{id}/documents
Then all files are saved and individual license_documents records are created for each
And the response includes the list of created document records
```

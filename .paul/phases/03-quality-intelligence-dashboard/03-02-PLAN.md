<objective>
## Goal
Add analytics charts, filterable issue browser, and semantic search to the Quality module — transforming the dashboard from a static summary into an interactive intelligence tool.

## Purpose
Plan 01 delivered the data surface (stats + tables). Plan 02 adds the *intelligence* layer: visual breakdowns that reveal patterns across projects, a filterable browse page for drilling into issues, and semantic search powered by the vector DB built in Phase 2. This is what makes the quality module a decision-making tool, not just a data viewer.

## Output
- Chart.js analytics on the dashboard (issues by type, status, trade, severity)
- New `/quality/browse` page with filterable/searchable issue table
- Semantic search endpoint + UI using existing vectordb infrastructure
- New aggregation API endpoints for chart data
</objective>


    `GET /quality/api/by-status` — returns `[{"status": "open", "count": N}, ...]`
    ```python
    SELECT status, COUNT(*) as count FROM quality_issues qi
    WHERE 1=1 {bu_filter}
    GROUP BY status ORDER BY count DESC
    ```

    `GET /quality/api/by-trade` — returns `[{"trade": "Mechanical", "count": N}, ...]`
    ```python
    SELECT trade, COUNT(*) as count FROM quality_issues qi
    WHERE trade IS NOT NULL AND trade != '' AND 1=1 {bu_filter}
    GROUP BY trade ORDER BY count DESC
    ```

    **Semantic search endpoint:**

    `GET /quality/api/search?q=...` — semantic search with SQL fallback
    ```python
    @bp.route("/api/search")
    def api_search():
        query = request.args.get("q", "").strip()
        if not query:
            return jsonify([])

        # Try vector search first
        try:
            from qms.vectordb.search import search_collection
            results = search_collection("quality_issues", query, n_results=20)
            if results and results.get("ids") and results["ids"][0]:
                # Extract issue IDs from vectordb results
                issue_ids = [int(doc_id.split("_")[-1]) for doc_id in results["ids"][0]]
                # Fetch full issue data from DB
                ... (query quality_issues WHERE id IN issue_ids, preserve order)
                return jsonify(issues)
        except Exception:
            pass  # Fallback to SQL LIKE

        # SQL LIKE fallback
        with get_db(readonly=True) as conn:
            rows = conn.execute("""
                SELECT qi.*, p.number AS project_number, p.name AS project_name
                FROM quality_issues qi
                LEFT JOIN projects p ON p.id = qi.project_id
                WHERE (qi.title LIKE ? OR qi.description LIKE ?)
                {bu_filter}
                ORDER BY qi.created_at DESC LIMIT 50
            """, (f"%{query}%", f"%{query}%") + bu_params).fetchall()
        return jsonify([dict(r) for r in rows])
    ```

    ```python
    def browse():
        return render_template("quality/browse.html")
    ```
